01
GOOGLE SHEETS API · OPENAI · EMBEDDINGS · REVOPS

KINGS CADENCE

Cold email, minus the cold sweat.

A full outbound pipeline that starts from nothing but a Google Sheet of prospect names. For each company it runs a news screen — mergers, expansions, funding, fresh regulation — so the outreach hooks onto something that actually happened this quarter, not a generic pitch. That signal is embedded and matched against the report catalog, with GPT reranking the shortlist so the single most relevant report wins, not just the closest vector. It then drafts a four-touch advisory cadence — each email personalized to the company, the trigger event, and the matched report — and writes the whole sequence back into the sheet the team already lives in. No new tab, no new tool to learn, no copy-pasting between systems. The sales team wakes up to a finished cadence and takes the credit. I'm at peace with this.

02
EMBEDDINGS · GPT RERANK · 3,000+ REPORT CATALOG

REPORT MAPPER

3,000 reports. One right answer per lead.

Purchased B2B lists have two chronic diseases: irrelevant matches and stale job data. This fixes both in one pass over a 3,000+ report catalog. Matching is two-stage by design — embeddings do the cheap, fast shortlist across the whole catalog, then GPT reranks only the top candidates, so you get semantic accuracy without paying LLM prices on every comparison. Before a match is trusted, the tool verifies the lead still works where the list claims: people change jobs constantly, and an email about their old employer's industry is worse than no email. When it detects a move, it finds the new employer and quietly rematches the lead against the catalog under the new context. What comes out is a list where every row has a defensible reason to exist.

PROSPECT LIST→ SHORTLIST→ RERANK→ STILL EMPLOYED?→ MATCH ✓

03
SEO DIAGNOSTICS · SEARCH CONSOLE API · BULK CSV

RANKING ANALYSER

An X-ray machine for search rankings.

Point it at one of your URLs and it does what an SEO consultant does in a two-hour audit: finds the competing page each rival ranks for the same intent, tears both down side by side, and scores the gap across weighted factors — content depth, schema coverage, in-content link structure, E-E-A-T signals, and whether AI overviews are quoting them instead of you. Every factor carries an explicit weight, so the output is a ranked diagnosis of *why* they win, not "content is king" hand-waving. The verdict comes back as actual numbers you can prioritize fixes against. Bulk mode scales the same X-ray to an entire CSV of URLs, enriching each row with live rank, CTR, impressions and clicks pulled straight from the Search Console API — one upload, a full portfolio health report out.

YOUR URL→ PULL COMPETITORS→ WEIGHTED DIAGNOSIS→ WHY THEY WIN ✓

04
OPENAI API · SERVERLESS · VERCEL · SEO

KR INTERLINKER

Internal links that place themselves.

Internal linking is the job everyone agrees matters and nobody wants to do: reread the whole library, remember what exists, find natural anchor text, repeat for every new post. This automates the entire loop. It reads the content library, understands what each piece is actually about via the OpenAI API, and proposes contextual link placements — real anchor text in real sentences, pointing at the genuinely related piece, not keyword-stuffed shortcuts. Architecture is a strict frontend/backend split deployed serverless on Vercel: the browser only ever talks to my API layer, and the OpenAI key lives server-side in environment config, because "we got scraped" is not a fun thing to say in standup. Built for Kings Research and running against the production content library, not a demo dataset.
